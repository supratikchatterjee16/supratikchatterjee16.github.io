News Analysis

What if we could get the sequence of events that have been reported in the news for any current incident or for any entity ever mentioned in news?


This was a question that occured to me about back in 2019. I decided to create a prototype for the same, as at that time, there were no other work related to it. These prototypes that I created, were called NASTY and NASTY UI.
While working on it there was a paper I released an Academic Paper titled, Entity Analysis and Centralization.
Currently, the same has been broken into several segments to make every component robust. Why was it necessary? The prototype that was created was a one shot prototype. If you run it, there is no way to stop it and if it has errors, you need to re-run it again. It was made to not fetch URLs that had been already travelled or fetched within a time-duration. Apart from this there were several other problems. For example Stanford Core NLP, which makes use of Statistics based NLP, which in our scenario caused a lot of problems.
What is being done about it now? The following is a list of current components being worked upon :
  1. serp_bot
    Python library required for modifying User Agent strings for a layer of bot detection prevention
  2. project-nlp
    Program written in Rust to analyze words to perform General Grammatical analysis of words.
  3. news_raker
    Crawlers for RSS feed and HTML page scraping. This is responsible for storing the extracted passages.
These components together provide a much better process for performing the same, while allowing us to make changes to further improve and optimize the components for this.
